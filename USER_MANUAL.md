# 语音笔记 — 用户手册

## 概述

语音笔记是一款 Android 录音与智能整理 App。录制或导入音频后，自动将语音转为文字（ASR），再通过 AI 生成结构化总结：议题、结论、待办事项和跟进计划。

**核心特色**：语音转写和 AI 总结均支持**在线**和**离线**两种模式，可独立切换。无需网络也能完成完整的录音→转写→总结流程。

---

## 一、主界面

打开 App 后进入**首页**，顶部红/蓝色标题栏显示"语音笔记"。

| 区域 | 说明 |
|---|---|
| 今日录音 / 今日时长 | 当天统计概览 |
| 最近录音 | 按时间倒序展示录音卡片，已生成总结的卡片会显示蓝色「已总结」标签 |
| 📜 右上角 | 进入历史记录 |
| ⚙ 右上角 | 进入设置 |
| ⊕ 右上角 | 新建录音 |

点击任意录音卡片进入**详情页**。

---

## 二、新建录音

点击首页右上角的 **⊕** 按钮，进入新建录音页面。

### 填写信息（全部可选）

- **标题**：录音名称，留空则自动生成（格式："新录音 6月25日 14:30"）
- **备注**：补充说明，会显示在历史卡片副标题
- **说话人**：逗号分隔的参与人员姓名

### 开始录音

点击 **开始录音** 按钮，App 进入录音状态：

- 状态栏出现录音通知（Android 前台服务）
- 界面顶部变红，显示脉冲红点 + 计时器
- 下方区域实时滚动显示语音转写文本
- 锁屏或切换到其他 App 时录音不会中断

### 结束录音

点击底部红色 **结束录音** 按钮，App 自动：

1. 停止录音并保存音频文件
2. 完成语音转写（如实时转写失败，会自动用离线文件方式重试）
3. 调用 AI 生成结构化总结
4. 跳转回首页

### 导入音频

如果已有录音文件，可点击 **导入音频** 按钮，从手机存储中选择音频文件。导入后会启动后台转写和总结，完成后在首页和详情页可查看。

---

## 三、录音详情

从首页或历史记录点击录音卡片进入详情页。页面顶部有三个 Tab：

### 3.1 音频

- 显示录音基本信息（标题、备注、时间、时长、说话人）
- **音频回放**：播放/暂停、快退 15 秒、快进 15 秒、拖动进度条
- 右上角可分享导出音频文件

### 3.2 转写

显示语音转写文本的状态和内容：

| 状态 | 说明 |
|---|---|
| 转写准备中... | 等待转写开始 |
| 正在转写... | 转写进行中（可取消） |
| 转写失败 | 显示错误信息，可点击重试 |
| 文件名.txt ✓ | 转写完成，点击查看全文，可导出分享 |

转写失败或不满意时可点击 **重新转写**。

### 3.3 总结

显示 AI 结构化总结：

- **议题**（蓝色标签）：本次录音讨论的主要话题
- **结论**（紫色标签）：达成的共识或决定
- **待办**（红色标签）：任务内容、负责人、截止时间
- **后续**（绿色标签）：下一步跟进计划

总结失败时可以 **重新总结**，完成后可 **导出** 分享。

## 四、设置

首页右上角 ⚙ 进入设置页。ASR（语音转写）和 LLM（AI 总结）各自独立配置。

### 4.1 语音识别

通过**在线/离线开关**切换模式：

#### 在线模式

填写 FunASR WebSocket 地址，例如：`ws://192.168.1.100:10095`

前提条件：需要自行部署 FunASR 服务端（不在 App 范围内）。

#### 离线模式

无需任何网络，语音识别在本地完成。

**选择模型质量**：INT8（推荐，约 170MB）或 FP32（精度更高，约 860MB）。

> ⚠️ 设备内存低于 4GB 不建议使用 FP32。

**下载模型**：点击「下载」按钮，App 从网络自动下载（约需几分钟）。

也可以先用电脑下载好，传到手机上后点「上传」导入：

| 模型 | 下载地址 |
|---|---|
| ASR INT8 (~170MB) | `https://github.com/k2-fsa/sherpa-onnx/releases/download/asr-models/sherpa-onnx-sense-voice-zh-en-ja-ko-yue-int8-2025-09-09.tar.bz2` |
| ASR FP32 (~860MB) | `https://github.com/k2-fsa/sherpa-onnx/releases/download/asr-models/sherpa-onnx-sense-voice-zh-en-ja-ko-yue-2025-09-09.tar.bz2` |

> 也可以用项目提供的脚本下载：`bash scripts/download_models.sh int8`

下载完成后显示 "模型已就绪" 绿色提示。

### 4.2 AI 总结

同样通过**在线/离线开关**切换模式：

#### 在线模式

填写以下信息：

| 配置项 | 说明 | 示例 |
|---|---|---|
| API 地址 | OpenAI 兼容的 base URL | `https://api.deepseek.com` |
| API Key | 服务商提供的密钥 | `sk-xxxxxxxx` |
| 模型 | 模型名称 | `deepseek-v4-pro`、`gpt-4o-mini` |

支持的 API 服务商：DeepSeek、OpenAI、Ollama（`http://localhost:11434`）等任何兼容 OpenAI 接口的服务。

> 长文本（超过 6000 字）会自动分段总结再合并，无需担心超长录音。

#### 离线模式

无需任何网络，AI 总结在本地完成。

**选择模型**：

| 模型 | 大小 | 适用场景 |
|---|---|---|
| Qwen2.5-0.5B | ~352 MB | 低内存设备（< 3GB），速度快 |
| Qwen2.5-1.5B | ~986 MB | 内存充足设备，质量更好 |
| 自定义模型 | — | 自行导入其他 GGUF 格式模型 |

**下载/上传模型**：

- 点击「下载」从 ModelScope 自动下载 GGUF 模型文件
- 点击「上传」从本地导入 GGUF 文件

也可用电脑提前下载，传到手机上后点「上传」导入：

| 模型 | 下载地址 |
|---|---|
| Qwen2.5-0.5B (~352MB) | `https://modelscope.cn/models/qwen/Qwen2.5-0.5B-Instruct-gguf/resolve/master/qwen2.5-0.5b-instruct-q4_k_m.gguf` |
| Qwen2.5-1.5B (~986MB) | `https://modelscope.cn/models/Qwen/Qwen2.5-1.5B-Instruct-GGUF/resolve/master/qwen2.5-1.5b-instruct-q4_k_m.gguf` |

> 低内存设备（< 3GB RAM）会自动使用 CPU-only 推理，避免 GPU 内存不足导致崩溃。

> 收到系统内存警告时，模型会在当前推理完成后自动释放。

### 4.3 保存

配置完成后务必点击 **保存** 按钮，设置才会持久化。保存成功会弹出摘要提示。

### 4.4 连接测试

在线模式下可点击「测试连接」，检测 ASR WebSocket 和 LLM API 是否可达。

## 五、历史记录

首页右上角 📜 进入历史记录页。

- **搜索**：输入关键词，按标题、备注、描述、转写文本模糊匹配
- **侧滑删除**：左滑录音卡片，出现红色删除区域，松开即删除
- **清空全部**：右上角垃圾桶图标，确认后删除所有记录及对应音频文件

## 六、常见场景速查

| 场景 | ASR 模式 | LLM 模式 | 需准备 |
|---|---|---|---|
| 完全离线使用 | 离线 | 离线 | 下载 ASR 模型 + LLM 模型 |
| 公司内网环境 | 在线 | 在线 | FunASR 服务地址 + LLM API 地址/Key |
| 离线录音 + 在线总结 | 离线 | 在线 | ASR 模型 + LLM API Key |
| 在线转写 + 离线总结 | 在线 | 离线 | FunASR 服务地址 + LLM 模型 |

---

## 七、注意事项

- 录音过程中请勿强制关闭 App，应通过「结束录音」按钮正常停止
- 离线 ASR 模型约 170~860MB，下载请使用 Wi-Fi
- 离线 LLM 推理依赖设备性能，0.5B 模型普通手机约需 30~60 秒，1.5B 模型约需 1~3 分钟
- 长时间录音（> 1 小时）会触发低电量提醒
- 删除录音记录会同时删除音频文件和转写文件，请确认后操作
